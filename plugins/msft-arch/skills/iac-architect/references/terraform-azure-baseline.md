# Terraform Azure Baseline

Canonical provider config, version constraints, backend setup, directory layout, and naming conventions for every Terraform project targeting Azure.

## Provider and Version Pinning

Pin Terraform core and both Azure providers. AVM module specs (TFFR3) mandate `azurerm ~> 4.0` and `azapi ~> 2.0`.

```hcl
terraform {
  required_version = "~> 1.9"

  required_providers {
    azurerm = {
      source  = "hashicorp/azurerm"
      version = "~> 4.0"
    }
    azapi = {
      source  = "Azure/azapi"
      version = "~> 2.0"
    }
  }

  backend "azurerm" {
    resource_group_name  = "rg-tfstate-prod"
    storage_account_name = "satfstate<suffix>"   # globally unique; use random suffix
    container_name       = "tfstate"
    key                  = "<project>/<env>/terraform.tfstate"
    use_azuread_auth     = true                  # RBAC over access keys
  }
}

provider "azurerm" {
  features {}
  use_oidc = true   # for CI; falls back to az CLI locally
}

provider "azapi" {}
```

`use_azuread_auth = true` forces the backend to authenticate via Entra ID (RBAC) rather than a storage access key. Grant the pipeline's Managed Identity or OIDC service principal the `Storage Blob Data Contributor` role on the container.

## Naming Conventions

Follow the Cloud Adoption Framework (CAF) naming guidance. Use hyphens as word separators; no underscores in Azure resource names (underscores are reserved for Terraform resource labels).

| Resource type | Pattern | Example |
|---|---|---|
| Resource Group | `rg-<workload>-<env>-<region>` | `rg-orders-prod-eus2` |
| Storage Account | `sa<workload><env><suffix>` | `saordersproda3k9` |
| Key Vault | `kv-<workload>-<env>-<region>` | `kv-orders-prod-eus2` |
| App Service | `app-<workload>-<env>-<region>` | `app-orders-prod-eus2` |
| AKS Cluster | `aks-<workload>-<env>-<region>` | `aks-orders-prod-eus2` |
| VNet | `vnet-<workload>-<env>-<region>` | `vnet-orders-prod-eus2` |

Encode region as a short code (`eus2` = East US 2, `weu` = West Europe). Store the mapping in a `locals.tf` using `avm-utl-regions` to avoid hand-coding region codes.

## Mandatory Resource Tags

Apply to every resource via a provider-level `default_tags` block (azurerm provider) or a `locals`-based tag map passed to every module:

```hcl
locals {
  tags = {
    cost-center = var.cost_center
    environment = var.environment
    owner       = var.owner
    project     = var.project
    expiry      = var.expiry   # ISO date or "permanent"
  }
}
```

Missing tags in a PR fail the OPA/Conftest gate; see `policy-as-code.md`.

## Variable Hygiene

- Secrets (connection strings, API keys) are NEVER in `.tfvars` or state. Use Key Vault data sources:

```hcl
data "azurerm_key_vault_secret" "db_password" {
  name         = "db-password"
  key_vault_id = azurerm_key_vault.main.id
}
```

- Mark outputs that contain derived secrets as `sensitive = true`.
- CI authenticates to Azure via OIDC (federated credential); no `ARM_CLIENT_SECRET` in pipeline variables.
