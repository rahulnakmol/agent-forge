## Register for Self-Service Password Reset process flow

> **Architect Notes**: This is the process flow for Register for Self-Service Password Reset. Each of the yellow boxes is a functional role and the tasks they perform are represented by green boxes on the right. These swim lanes make it easy to see who is responsible for each task. Tasks and processes are indexed so that you can easily see where they fit in the overall taxonomy. They are also linked by gateway markers that indicate if a task is performed in parallel or in sequence. The task boxes have a lot of information built in. ## Role Based Process Flows – Tasks

> **Architect Notes**: A red CP at the top right would indicate that this has a control point. A red O in the bottom left means that this activity is performed manually in the software system, and a blue “A” indicates that the activity is automated. This red M indicates that the task is performed outside of the system manually. A teal R on the top left would indicate that there is a report associated with this activity.
