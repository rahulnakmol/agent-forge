## Environment Architecture

> **Architect Notes**: The Environment Architecture is a tech-specific asset and environment usage plan and timeline that identifies the necessary type and number of environments required to support the software development lifecycle. It also shows how they will be used or updated before, during and after deployment. This asset details the reference set of environments that recommends for a technology platform and shows environment usage and migrations during and after the project lifecycle. The Environment Architecture defines all environments used to support build and deployment, when they will be needed, what will be in them at any stage during the software development lifecycle, what will be migrated between environments and when, and how they will be used post go-live. This is an example of the Environment Architecture plan tailored for an Oracle solution. On the left side, you'll find details related to the system, domain, environment, and its intended use. On the right, you'll see the implementation timeline for each environment.
