# Category-Specific Question Banks

## Phase 2 Question Reference

Use these question banks during Phase 2 (Context & Constraints Gathering). Select questions appropriate to the classified task category from Phase 1. Batch 2-3 related questions per AskUserQuestion call.

---

## Universal Questions (All Categories)

Always ask these regardless of category:

1. **Audience**: Who is the audience or consumer of this output? What's their technical level and familiarity with the subject?
2. **Detail level**: What level of detail do you need — high-level overview, moderate depth, or exhaustive detail?
3. **Constraints**: Are there any constraints I should know about — deadlines, word limits, format requirements, existing content to incorporate, or organizational guidelines?

---

## Artifact: Document (DOCX / MD)

**Structure**:
- What structure or sections do you envision? (e.g., executive summary, background, analysis, recommendations)
- Is there a specific page count or word limit target?
- Should it include a table of contents, headers/footers, or page numbers?

**Content**:
- Do you have any reference materials, templates, or examples I should follow?
- Is there existing content that should be incorporated or built upon?
- Are there specific data points, quotes, or references that must be included?

**Brand & Guidelines**:
- What brand or organizational guidelines apply? (e.g., company style guide, WTW, internal brand standards)
- Is there a specific template or letterhead to use?
- What formatting conventions does your organization follow?

---

## Artifact: Spreadsheet (XLSX)

**Structure**:
- What sheets or tabs do you need? What should each contain?
- What are the key columns and data fields?
- Do you need formulas, pivot tables, or charts?

**Data**:
- Do you have source data to incorporate, or should I create a template with sample data?
- What calculations or transformations need to happen?
- Are there lookup tables or reference data to include?

**Usage**:
- Who will use this spreadsheet and for what purpose?
- Does it need to support ongoing data entry or is it a one-time analysis?
- Should it include data validation, conditional formatting, or protection?

---

## Artifact: Presentation (PPTX)

**Structure**:
- How many slides do you envision?
- What's the narrative arc? (problem → solution → results, chronological, thematic, etc.)
- Do you need speaker notes for each slide?

**Content**:
- What key messages must come through?
- Are there specific data visualizations, charts, or diagrams needed?
- Do you have any images, logos, or brand assets to include?

**Context**:
- What's the setting? (board meeting, conference talk, team all-hands, client pitch)
- How long is the presentation time slot?
- Will it be presented live or shared asynchronously?

---

## Artifact: Markdown (MD)

**Structure**:
- What type of Markdown document? (README, technical spec, blog post, documentation, tutorial)
- What sections or headings do you need?
- Should it include code blocks, tables, or diagrams (Mermaid)?

**Content**:
- What's the primary purpose — inform, instruct, persuade, or document?
- Are there API references, configuration examples, or code samples to include?
- Should it link to external resources or other documentation?

**Platform**:
- Where will this Markdown be rendered? (GitHub, Notion, static site, internal wiki)
- Are there any platform-specific formatting considerations?

---

## Code

**Stack & Context**:
- What tech stack, language, or framework are you using?
- Is this greenfield or modifying existing code? If existing, where in the codebase?
- What's the runtime environment (Node.js, Python, .NET, browser, etc.)?

**Requirements**:
- What specific functionality should the code implement?
- Are there API contracts, data models, or interfaces it must conform to?
- What edge cases or error handling should be addressed?

**Quality**:
- What testing expectations do you have? (unit tests, integration tests, coverage targets)
- Are there linting, formatting, or style guide requirements?
- Should the code include comments, documentation, or type annotations?

**Integration**:
- How does this code fit into the broader system?
- What existing modules, services, or databases does it interact with?
- Are there performance requirements or constraints?

---

## Research

**Scope**:
- How current does the information need to be? (latest only, historical context included, specific date range)
- Should I prioritize breadth (many sources/perspectives) or depth (detailed analysis of fewer topics)?
- What geographic or industry scope applies?

**Sources & Perspectives**:
- Are there specific sources, publications, or experts to include or exclude?
- Should the research present a balanced view or argue a specific position?
- Do you need citations or references in a specific format?

**Output**:
- What format should the research take? (summary brief, annotated bibliography, comparative analysis, report)
- What level of evidence quality is expected? (peer-reviewed only, industry reports acceptable, primary sources required)

---

## Query / Answer

**Depth**:
- How technical should the explanation be? (beginner-friendly, intermediate, expert-level)
- Do you want a single definitive answer or a comparison of perspectives/approaches?
- Should I include examples, analogies, or visualizations?

**Context**:
- What's your existing knowledge level on this topic?
- Is this for your own understanding or are you preparing to explain it to someone else?
- Are there specific aspects or sub-questions you want addressed?

---

## Workflow / Automation

**Scope**:
- What's the end-to-end process you want to set up?
- What triggers the workflow and what's the final output?
- Are there manual approval steps or is it fully automated?

**Environment**:
- What tools and platforms are involved? (CI/CD, cloud services, APIs, databases)
- What access credentials or permissions are needed?
- Are there existing workflows to integrate with or replace?

**Reliability**:
- What happens when a step fails? (retry, alert, fallback, stop)
- What monitoring or logging is needed?
- What are the SLA or uptime expectations?
